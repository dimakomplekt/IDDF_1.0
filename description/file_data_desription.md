# Описание макроса

Макрос, совместно с макросом заполнения рамок, предназначен для автоматического заполнения конструкторской документации (КД) за счёт передачи данных в параметры моделей (деталей и сборок) из общей ведомости (Excel).

**Макрос работает только при использовании стандартной структуры дерева моделей организации.**

После формирования структуры изделия создаётся единая ведомость состава изделия.
В ней каждому элементу присваиваются:

* обозначение (шифр)
* наименование
* позиционный номер
* признаки типа (сборка / деталь)
* статус изготовления (производство / закупка)
* данные участников разработки и проверки
* даты выполнения этапов
* организация-изготовитель

Эти данные далее транслируются в параметры моделей (Inventor parameters), где используются для:

заполнения основной надписи чертежей
формирования спецификаций
автоматизации оформления КД

___


## Структура таблицы ведомости

Таблица ведомости состоит из строк пояснений и строк с данными элементами изделия (столбцы - виды данных).

Таблица заполняется начиная с главной сборки с разделением каждой подсборки (включая основную) на такие части, как

* Подсборки текущей сборки
* Детали текущей сборки
* Метизы (крепежные элементы) текущей сборки

После завершения части ведомости основной подсборки следует формирование части ведомости первой по списку подсборки внутри основной сборки с переходом к следующей (ко второй) подсборки внутри основной сборки лишь после завершения вложенного формирования списков для всех подсборок первой по списку подсборки внутри основной сборки. Т.е., если основная сборка состоит из подсборки 1 и подсборки 2 и деталей, а подсборка 1 состоит из подсборок 1.1 и 1.2, то ведомость формируется, как:

* Элементы основной сборки
* Элементы подсборки 1
* Элементы подсборки 1.1
* Элементы подсборки 1.2
* Элементы подсборки 2

В ведомости присутствуют следующие основные данные о моделях (столбцы)

* **N Позиции** — номер элемента, соответствующий спецификации - **ячейки A**
* **Название модели** — полное имя модели, которое присваивается моделям в директориях - **ячейки B**
* **Обозначение** — код изделия по ведомости, соответствующий спецификации - **ячейки C**
* **Наименование**— краткое название по ведомости, соответствующее спецификации - **ячейки D**
* **Деталь / Сборка** — тип объекта (используется для формирования основной надписи) - **ячейки E**
* **Пр-во / Закуп** — способ получения изделия (закупные изделия игнорируются в присвоении данных) - **ячейки F**
* **Разраб. — разработчик** - кто делал модель - **ячейки G**
* **Дата проверки 1** — дата разработки (когда завершил делать модель) - **ячейки H**
* **Пров. — проверяющий** - **ячейки I**
* **Дата проверки 2** — дата завершения проверки  - **ячейки J**
* **Т. контр — техконтроль**  - **ячейки K**
* **Дата проверки 3** — дата завершения техконтроля  - **ячейки L**
* **Нач. отд. — начальник отдела**  - **ячейки M**
* **Дата проверки 4** — дата завершения утверждения отделом  - **ячейки N**
* **Н. контр. — нормоконтроль** - **ячейки O**
* **Дата проверки 5** — дата завершения нормоконтроля - **ячейки P**
* **Утв.** — утверждающий - **ячейки Q**
* **Дата проверки 6** — дата завершения утверждения - **ячейки R**
* **Компания** — организация (либо которая разработала изделие, либо которая будет изготавливать изделие) - **ячейки S**

Они формируют подобную таблицу

Все графы, кроме графы **"Название модели"** формируются в ведомости вручную. Графа название модели формируется слиянием ячеек "Обозначение" и "Наименование" с вставкой между ними строки " - " для производимых изделий и дублированием ячейки "Наименование" для закупных изделий. Всем названиям в конец строки добавляется обозначение формата, исходя из того,  является ли модель сборкой или деталью (".iam" или ".ipt").

## Алгоритм передачи данных

### Основные сведения

* Макрос работает **только** при использовании стандартного дерева моделей организации вида:

```

├───N_Модели
│   ├───0_Архив
│   ├───1_Детали
│   └───2_Подсборки
│       ├───1_Детали
│       └───2_Подсборки
|
| ...
|

```

* Макрос хранится в локальных правилах iLogic файла главной сборки изделия, размещенного в корневой папке (N_Модели) дерева моделей.

* Макрос запускается из файла главной сборки изделия, размещенного в корневой папке (N_Модели) дерева моделей.

* Таблица ведомости состава изделия размещается той же корневой папке (N_Модели) дерева моделей (название - "Ведомость состава изделия.xlsx").

* При **изменении данных **в таблице ведомости, **актуализации имён файлов** и **повторном запуске макроса** все модели и чертежи получают актуальные данные. Отображение данных на чертежах обновляется, согласно новым данным.


### Алгоритм работы макроса

0) Обнаруживаем таблицу ведомости в папке вызова файла по стандартному названию таблицы - "Ведомость состава изделия.xlsx"

1) Начиная с общей сборки рекурсивно идём по дереву моделей любой вложенности 

    1.1. Для каждой (кроме уже просмотренных) детали или подсборки парсим данные

        1.1.1 Проверяем совпадения имени файла сборки или детали со значениями ячеек имени файла (все ячейки с индексом столбца B). При обнаружении полного совпадения фиксируем номер строки

            1.1.1.1. Проверяем статус ячейки "Пр-во / Закуп"
            
                - При статусе "Закуп" — выходим из текущей модели, не сохраняя изменения, переходим обратно в пункт 1.1  
                - При статусе "Пр-во" — переходим к следующему шагу вложенного цикла (1.1.1.2)
                
            1.1.1.2. Создаем (или меняем содержимое существующих) в файле текущего открытого чертежа или подсборки пользовательские текстовые параметры и заполняем их данными из таблицы:

                - "position_number" — номер элемента (позиционное обозначение), соответствующий спецификации - из ячейки A строки найденной в пункте 1.1.1
                - "part_number" — код изделия по ведомости, соответствующий спецификации - из ячейки C строки найденной в пункте 1.1.1
                - "part_name" — краткое название по ведомости, соответствующее спецификации - из ячейки D строки найденной в пункте 1.1.1
                - "part_type" — тип объекта (используется для формирования основной надписи) - из ячейки E строки найденной в пункте 1.1.1
                - "part_developer" — разработчик (кто делал модель) - из ячейки G строки найденной в пункте 1.1.1
                - "developer_date" — дата разработки (когда завершил делать модель) - из ячейки H строки найденной в пункте 1.1.1
                - "part_test" — проверяющий - из ячейки I строки найденной в пункте 1.1.1
                - "test_date" — дата завершения проверки - из ячейки J строки найденной в пункте 1.1.1
                - "part_tech_control" — техконтроль - из ячейки K строки найденной в пункте 1.1.1
                - "tech_control_date" — дата завершения техконтроля - из ячейки L строки найденной в пункте 1.1.1
                - "part_department_head" — начальник отдела - из ячейки M строки найденной в пункте 1.1.1
                - "department_head_date" — дата утверждения отделом - из ячейки N строки найденной в пункте 1.1.1
                - "part_norms_control" — нормоконтроль - из ячейки O строки найденной в пункте 1.1.1
                - "norms_control_date" — дата нормоконтроля - из ячейки P строки найденной в пункте 1.1.1
                - "part_approved_by" — утверждающий - из ячейки Q строки найденной в пункте 1.1.1
                - "part_approved_date" — дата утверждения - из ячейки R строки найденной в пункте 1.1.1
                - "part_company" — организация (разработчик или производитель) - из ячейки S строки найденной в пункте 1.1.1
  
            1.1.1.3. Дублируем некоторые из полученных данных на шаге 1.1.1.2. в параметры текущего открытого документа, используя присвоение свойств iProperties

                - "position_number" — iProperties.Value("Design Tracking Properties", "Stock Number")
                - "part_number" — iProperties.Value("Design Tracking Properties", "Part Number")
                - "part_name" —  iProperties.Value("Inventor Summary Information", "Title")
                - "part_developer" — iProperties.Value("Inventor Summary Information", "Author")
                - "developer_date" — iProperties.Value("Design Tracking Properties", "Creation Time")
                - "part_test" —  iProperties.Value("Design Tracking Properties", "Checked By")
                - "test_date" —  iProperties.Value("Design Tracking Properties", "Date Checked")
                - "part_tech_control" - iProperties.Value("Design Tracking Properties", "Engineer")
                - "part_department_head" — iProperties.Value("Inventor Document Summary Information", "Manager")
                - "part_norms_control" —  iProperties.Value("Design Tracking Properties", "Engr Approved By")
                - "norms_control_date" —  iProperties.Value("Design Tracking Properties", "Engr Date Approved")
                - "part_approved_by" —  iProperties.Value("Design Tracking Properties", "Mfg Approved By")
                - "part_approved_date" —  iProperties.Value("Design Tracking Properties", "Mfg Date Approved")
                - "part_company" —  iProperties.Value("Inventor Document Summary Information", "Company")

            1.1.1.4. Сохраняем изменения, закрываем файл детали или сборки (КРОМЕ ОБЩЕЙ СБОРКИ ИЗ КОТОРОЙ БЫЛ ВЫЗВАН МАКРОС).
                

2) При завершении обхода без проблем сигнализируем об успешном внесении изменений и выводим лог - какие данные были присвоены каким деталям (лог собирается последовательно на каждом шагу), формат лога:

```
ПРОПУСК СТРОКИ
---------------------------------------------------------
"Имя документа"
---------------------------------------------------------
Присвоенные свойства:
ПРОПУСК СТРОКИ
    - position_number: "текущее значение"
    - part_number: "текущее значение"
    - part_name: "текущее значение"
    - part_type: "текущее значение"
    - part_developer: "текущее значение"
    - developer_date: "текущее значение"
    - part_test: "текущее значение"
    - test_date: "текущее значение"
    - part_tech_control: "текущее значение"
    - tech_control_date: "текущее значение"
    - part_department_head: "текущее значение"
    - department_head_date: "текущее значение"
    - part_norms_control: "текущее значение"
    - norms_control_date: "текущее значение"
    - part_approved_by: "текущее значение"
    - part_approved_date: "текущее значение"
    - part_company: "текущее значение"
```
