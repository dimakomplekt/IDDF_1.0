# Описание макроса

Макрос, совместно с макросом получения параметров, предназначен для автоматического заполнения конструкторской документации (КД) за счёт использования переданных в параметры моделей (деталей и сборок) из общей ведомости (Excel) данных.

**Макрос работает только при использовании стандартной структуры дерева моделей организации.**

Макрос присваивает параметрам основной надписи чертежа требуемые параметры модели.

___


## Алгоритм передачи данных


### Основные сведения

* Макрос хранится в локальных правилах iLogic каждого чертежа.

* Макрос запускается из файла чертежа.

* При **изменении данных **в таблице ведомости, **актуализации имён файлов** и **повторном запуске макроса** все модели и чертежи получают актуальные данные. Отображение данных на чертежах обновляется, согласно новым данным.


### Алгоритм работы макроса

1) Получаем свойства и параметры основного файла детали или сборки для данного файла чертежа (через ActiveSheet.DrawingViews(1)).

2) Присваиваем: 

    Значения параметров из данного списка для основной модели (имя модели совпадает с именем файла чертежа - "<Основная или присоединенная модель> ..."):
   
    - "part_number" — код изделия по ведомости, соответствующий спецификации - из ячейки B строки найденной в пункте 1.1.1
    - "part_name" — краткое название по ведомости, соответствующее спецификации - из ячейки C строки найденной в пункте 1.1.1
    - "part_type" — тип объекта (используется для формирования основной надписи) - из ячейки D строки найденной в пункте 1.1.1
    - "part_developer" — разработчик (кто делал модель) - из ячейки F строки найденной в пункте 1.1.1
    - "developer_date" — дата разработки (когда завершил делать модель) - из ячейки G строки найденной в пункте 1.1.1
    - "part_test" — проверяющий - из ячейки H строки найденной в пункте 1.1.1
    - "test_date" — дата завершения проверки - из ячейки I строки найденной в пункте 1.1.1
    - "part_tech_control" — техконтроль - из ячейки J строки найденной в пункте 1.1.1
    - "tech_control_date" — дата завершения техконтроля - из ячейки K строки найденной в пункте 1.1.1
    - "part_department_head" — начальник отдела - из ячейки L строки найденной в пункте 1.1.1
    - "department_head_date" — дата утверждения отделом - из ячейки M строки найденной в пункте 1.1.1
    - "part_norms_control" — нормоконтроль - из ячейки N строки найденной в пункте 1.1.1
    - "norms_control_date" — дата нормоконтроля - из ячейки O строки найденной в пункте 1.1.1
    - "part_approved_by" — утверждающий - из ячейки P строки найденной в пункте 1.1.1
    - "part_approved_date" — дата утверждения - из ячейки Q строки найденной в пункте 1.1.1
    - "part_company" — организация (разработчик или производитель) - из ячейки R строки найденной в пункте 1.1.1

    параметрам из данного списка для текущего файла чертежа "<Текущий чертеж> ...":

    - iProperties.Value("Design Tracking Properties", "Part Number") = "part_number" пользовательских свойств основной детали или сборки
    - iProperties.Value("Inventor Summary Information", "Title") = "part_name" пользовательских свойств основной детали или сборки
    - iProperties.Value("Свойства ГОСТ", "Тип документа") - если "part_type" пользовательских свойств основной детали или сборки = "Сборка" - присваиваем "Сборочный чертеж", иначе - ""
    - iProperties.Value("Inventor Summary Information", "Author") = "part_developer" пользовательских свойств основной детали или сборки
    - iProperties.Value("Design Tracking Properties", "Creation Time") = "developer_date" — пользовательских свойств основной детали или сборки
    - iProperties.Value("Design Tracking Properties", "Checked By") = "part_test" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Date Checked") = "test_date" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Engineer") = "part_tech_control" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Свойства ГОСТ", "Технолог Дата") = "tech_control_date" - пользовательских свойств основной детали или сборки 
    - iProperties.Value("Inventor Document Summary Information", "Manager") = part_department_head"  — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Authority") = part_department_head" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Свойства ГОСТ", "Руководитель Дата") = "department_head_date"  — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Engr Date Approved") = "part_norms_control" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Mfg Approved By") = "norms_control_date" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Mfg Approved By") = "part_approved_by" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Design Tracking Properties", "Mfg Date Approved") = "part_approved_date" — пользовательских свойств основной детали или сборки 
    - iProperties.Value("Inventor Document Summary Information", "Company") = "part_company" — пользовательских свойств основной детали или сборки

3) Сохраняем изменения и не закрываем файл чертежа.                

4) При завершении обхода без проблем сигнализируем об успешном внесении изменений и сообщение о присвоении параметров.
